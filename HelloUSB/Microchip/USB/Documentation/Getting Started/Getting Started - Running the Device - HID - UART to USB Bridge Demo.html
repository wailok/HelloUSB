Firmware:

To run this project, you will need to load the corresponding firmware into the devices. 

 

The source code for this demo is available in the “<Install Directory>\USB Device – HID – UART to USB Bridge\Firmware” directory.  In this directory you will find all of the user level source and header files as well as project and workspace files for each of the hardware platforms.  Find the project (*.mcp) or workspace (*.mcw) file that corresponds to the hardware platform you wish to test.  Compile and program the demo code into the hardware platform.  For more help on how to compile and program projects, please refer to the MPLAB® IDE help available through the help menu of MPLAB IDE (Help->Topics…->MPLAB IDE).

 

This demo defaults to using 115200 as the baud rate.  In order to change the baud rate of the example, go to the HardwareProfile header file that corresponds to the hardware platform in use.  In the RS232 configuration section of this header file is a BAUDRATE definition.  Change the value of BAUDRATE and rebuild the project.

 

Running the Demo:

This demo allows the device to appear like a Human Interface Device (HID) but act as a UART to USB bridge.  This demo will take data received over the UART and send it over the USB.  Likewise, data received on the USB will be sent over the UART. 

 

Because the HID class uses fixed size reports and in this application a variable size data payload is required, the first byte of each transmission, both from the USB and the UART needs to be size of the data payload.  The application then has 63 bytes of data that they are allowed in a single transmission. 

 

 

Connect the hardware both to the serial port on the computer (or other target device) and the USB port.  Launch the “<install directory>\USB Device - HID - UART to USB bridge\UART to USB HID bridge.exe” application.

 

 

Click the connect button to start a connection to the device.  If the device is successfully connected the “Connect” button will become disabled and the “Send Data” button will become enabled.  At this point open up a terminal program and connect to the serial port that the device is attached to at the baud rate specified in the HardwareProfile header described in the previous section.

 

Typing in the terminal window will result in the decimal value of the ASCII value for that key to be displayed in the PC application.  For example if ‘1’ is press then ‘49’ will get printed in the PC application window.  This corresponds to the ASCII value 0x31 (or 49 decimal) for the key ‘1’. 

 

Typing in the text box and pressing the ‘Send Data’ button will result in that data being set starting from the USB, through the UART, to the Terminal application.
